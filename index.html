HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • HTML code is written in VS

CSS

  • CSS is used to style the information on a website
  • This can be done using inline CSS, internal or external style sheets
  • Inline CSS and internal style sheets are quick to use within the HTML code
  • The downside of these is that you have to repeat the code for each instance
  • External style sheets are better used for repeated commands, such as how all headers should be formatted
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.

Git

  • git status: checks what branch we are currently on
  • git checkout -b "branch-name": creates a new branch and switches to it
  • git pull origin main: ensures you've pulled the main branch from its origin, i.e. Github, to the local computer
  • git add -A: commits all files in the working branch
  • git -m "message": adds a description to our commit, i.e. "add code to HTML file"

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • A selection of data within a container is called an array.
  • Data within an array is numbered sequentially, starting from 0
  • Common practice is to write x+1 as x++
  • Functions can be used to tell the computer how to perform a certain task.
  • It is not enough to simply define a function - it must be 'called' to be executed.
  • A function is called at the end of a script
  • Control flow is the order in which a computer executes code in a script.
  • Remember: order matters!
